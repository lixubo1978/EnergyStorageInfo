第一部分｜主要问题与解答

跳到文件定位 ↓

1.1 匈牙利调频/平衡产品类型：FCR / aFRR / mFRR（以及 RR/IGCC）

  • 在匈牙利的容量市场/数据体系里，FCR / aFRR / mFRR 是明确作为“容量市场名称”被记录与上报的（监管数据字段）。官方表格里直接有字段：
    “Kapacitás piac neve (FCR/aFRR/mFRR)”（容量市场名称）+ Kapacitás Ár (Ft/MW/h)（容量价）+ Tender megnevezése（招标名称）。（Magyar Közlöny）
  • 同一套官方口径里也区分了 aFRR/mFRR 的激活能量（aktivált kiegyenlítő szabályozási energia），并把它们作为需要报送的结算数据。（Magyar Közlöny）
结论：匈牙利至少同时存在（并被监管数据体系正式覆盖）FCR 容量采购、aFRR 容量/能量（激活）、mFRR 容量/能量（激活），且它们的价格口径包含容量价与能量价。

1.2 “里程/性能要求”如何落地：Akkreditáció（技术资质/测试）+ 计量/通信

核心：储能要进市场，到底要满足什么性能、要做哪些测试、要配哪些计量。

（A）MAVIR 的 Akkreditáció（技术资质/测试）是硬门槛

  • 多个匈牙利行业解读一致指出：参与系统服务/平衡服务前，MAVIR 会通过 akkreditáció / műszaki minősítés（技术认证/测试）确认技术能力，并且测试细节在《Üzemi Szabályzat（ÜSZ）》的 9.1 附件。（Voltrack）
  • 监管侧的数据体系里也单列了 “Akkreditációs eljárások（认证程序）”作为定期数据报送主题（它不是内部流程，而是监管关注项）。（net.jogtar.hu）

（B）计量/分表

  • 在官方《Elosztói Szabályzat（配网规则）》里明确写到：对于“被要求或希望提供 aFRR/mFRR 服务的用户物理单元与储能”，依据 Üzemi Szabályzat（ÜSZ）相关条款，需要为电能计量设置 “külön mérés（单独计量/独立电表）”。（mvmhalozat.hu）

（C）具体性能示例：mFRR 12.5 的“12.5 分钟”

  • 匈牙利行业材料明确写到：mFRR12,5 的“12.5”意味着从指令到目标值完全激活时间不得超过 12.5 分钟，并指出该细节在 ÜSZ 9.1 附件 7/a 点。（EQUANS Magyarország）
可落地结论：在匈牙利，要进 aFRR/mFRR（更不用说 FCR），不是“注册就能卖”，而是：
先满足计量/通信/控制等基础要求 → 走 MAVIR 的 Akkreditáció（ÜSZ 9.1 附件定义测试与要求）→ 才能参与容量/能量交易与结算。
其中 aFRR/mFRR/储能很大概率需要“单独计量（külön mérés）”。

1.3 调频价格：容量费 / 能量费 / 罚则（匈牙利口径怎么对应）

（1）容量费（Kapacitás Ár，Ft/MW/h）

  • 官方报送/表格字段明确：容量价单位是 Ft/MW/h，并按 FCR/aFRR/mFRR 分类记录，同时记录投标 MW、锁定 MW、招标名称等。（Magyar Közlöny）

（2）能量费（Kiegyenlítő energia ára，HUF/kWh）= 激活/平衡能量价格

  • 官方报送表格把 “Kiegyenlítő energia ára (HUF/kWh)”作为核心字段，并明确区分：Pozitív mFRR aktivált、Pozitív aFRR aktivált 等。（Magyar Közlöny）
  • 另一个官方报送模板“填表说明”还写到：报告中的能量价遵循 EBGL 的方向约定，并用于 MAVIR 的全国平衡能量结算。（njt.hu）

（3）罚则/扣款（非履约/不可用/偏差）怎么抓“匈牙利规则位置”

这一块的“硬条款”通常会落在 NÜKSZ（BSP/BRP 的规则章节）以及 MAVIR 的招标文件/合同条款里。

  • NÜKSZ 3.1.45：被引用为“mFRR 与 RR 的可用与激活规则”；激活按日内提交的能量价 merit order 执行，只有技术原因才可偏离。（Voltrack）
  • NÜKSZ 3.1.46 / 3.1.47：被引用为“mFRR 与 RR 容量费的结算规则”，并提到 MAVIR 会先给出预结算基础数据，最终按条款结算，且按月结算。（Voltrack）
可落地结论：匈牙利的调频收入至少两条线：
• 容量费（Ft/MW/h）：通过 MAVIR 的容量招标锁定；
• 能量费（HUF/kWh）：激活后按 aFRR/mFRR 实际能量结算；
• 罚则/扣款：通常由 NÜKSZ + MAVIR 合同/招标条款约束；关键定位点在 NÜKSZ 3.1.45–3.1.47（可用/激活/结算链条）。

第二部分｜具体官方文件及相关章节明细

证据链（从强到弱）：官方监管/官方规则 ＞ 官方模板 ＞ 行业引用定位。

A. 官方监管数据与模板

1）平衡/调频能量数据报送模板（含 aFRR/mFRR 激活能量与能量价 HUF/kWh）

  • 字段：“Kiegyenlítő energia ára (HUF/kWh)”
  • 字段：“Pozitív mFRR aktivált … / Pozitív aFRR aktivált …”
  • 用途：证明能量价口径与 aFRR/mFRR 激活分类在匈牙利是“官方口径”。（Magyar Közlöny）

2）容量市场数据报送模板（含 FCR/aFRR/mFRR、容量价 Ft/MW/h、投标与锁定容量、招标名称）

  • 字段：“Kapacitás piac neve (FCR/aFRR/mFRR)”
  • 字段：“Kapacitás Ár (Ft/MW/h)”
  • 字段：“Beadott pályázat MW / Lekötött kapacitás MW / Tender megnevezése”
  • 用途：证明容量费口径与产品拆分、以及招标维度在匈牙利是“官方口径”。（Magyar Közlöny）

3）1/2023 (IV.5) MEKH rendelet（MEKH 数据报送法令）

  • 条目：“Akkreditációs eljárások”等数据报送项（说明认证是监管关注的制度化事项）。（net.jogtar.hu）
  • 用途：作为内部报告里“监管制度依据”的锚点。

B. 官方电网规则（配网规则中的计量要求）

4）Elosztói Szabályzat（配网规则）— “单独计量（külön mérés）”要求

  • 条款点：9.8.2（对 aFRR/mFRR 服务（akkreditáció）义务或自愿提供者，需要满足 ÜSZ 条款并建设单独计量）。（mvmhalozat.hu）
  • 用途：这是 EPC/并网设计阶段必须落地的“工程要求”（否则后期补改成本很高）。

C. MAVIR / NÜKSZ / ÜSZ 的条款定位线索（待拿到原文后逐条核验）

5）Üzemi Szabályzat（ÜSZ）— 9.1 附件：Akkreditáció / 技术测试与要求

  • 行业引用一致指向：ÜSZ 9.1 Melléklet 包含 akkreditáció（技术认证/测试）细节。（Voltrack）
  • 后续下载到 ÜSZ PDF 原文后，应重点从 9.1 附件里提取：响应/激活时间、控制精度、通信链路、测试流程、合格标准、不合格处理等。

6）NÜKSZ — 3.1.45 / 3.1.46 / 3.1.47：激活顺序与容量费结算链条

  • 3.1.45：mFRR/RR 可用与激活；aFRR/mFRR 激活按能量报价排序（仅技术原因可偏离）。（Voltrack）
  • 3.1.46 / 3.1.47：mFRR/RR 容量费结算规则；MAVIR 预结算数据 → 最终结算；按月结算。（Voltrack）
提示：第二部分的“条款点/字段名/附件编号”用于尽调引用定位；后续若你更新到新版 PDF，建议额外标注版本号（发布日期/版本号），以保证引用可追溯。
